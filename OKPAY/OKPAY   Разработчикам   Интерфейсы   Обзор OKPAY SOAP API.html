Обзор OKPAY SOAP API

OKPAY SOAP API обеспечивает программный доступ к функциям и сервисам OKPAY.

Разработчики могут создавать собственные приложения, инструменты и услуги, которые соответствуют тем же сервисам и инструментам, имеющимся на главном сайте OKPAY, https://www.okpay.com/. Типичные области применения включают поиск по транзакциям, массовые платежи и возврат платежей. API базируется на открытых стандартах, известных под общим названием "Вэб Сервисы", которые включают Simple Object Access Protocol (SOAP), Web Services Definition Language (WSDL), и язык XML Schema Definition (XSD). Эти стандарты поддерживается широким спектром средств разработки на различных платформах.

OKPAY использует SOAP версии 1.1.

Адрес службы OKPAY

Для разработки приложений использующих OKPAY API требуется базовая схема и основные компоненты вэб-сервиса OKPAY. WSDL и XSD файлы размещены по адресу: https://api.okpay.com/OkPayAPI

Как работает SOAP API

Как и многие другие веб-сервисы, SOAP OKPAY представляет собой комбинацию клиентских и серверных схем, аппаратных средств и программного обеспечения серверов и базовых сервисов.

Рисунок 1. Диаграмма высокого уровня OKPAY SOAP

В объектно-ориентированной модели обработки, интерфейс к SOAP запросам/ответам — это объект на языке программирования вашего приложения. Ваше клиентское приложение генерирует SOAP интерфейсы бизнес-объектов и сетевые стабы (network stubs) из предоставленных (OKPAY) WSDL и XSD файлов, которые определяют структуру сообщений, их содержание и привязку к сервисам OKPAY API. SOAP приложение работает с данными в виде свойств объекта, т.е. передает и принимает данные вызывая методы объекта. SOAP клиент занимается формированием SOAP-запроса, отправляет его в сервис OKPAY и преобразует ответ обратно в объект.

Операции которые могут быть выполнены с помощью OKPAY API

API поддерживает следующие операции.

  • Общее. Универсальные функции: получить время сервера, проверить существует ли аккаунт в системе и т.д.
  • Платежи. Перевод средств с одного кошелька на другой. С помощью этих функций вы можете перевести деньги в любой из поддерживаемых OKPAY валют.
  • История. Запрос истории операций заданного кошелька. Поиск информациии о конкретных транзакциях. Могут использоваться дополнительные параметры, такие как указание диапазона дат, валюты, кошелька назначения и т.п.
  • Баланс. Запрос баланса. Получение баланса заданного кошелька во всех валютах или в конкретной валюте.

Аутентификация

Вызова API всегда должен содержать информацию, которая позволит серверу API аутентифицировать вызывающего пользователя. Вызов должен содержать ID кошелька и другие сведения, которые позволят серверу точно идентифицировать пользователя делающего запрос. Каждый API запрос должен содержать следующие параметры для проверки подлинности:

  • WalletID. Это ID вашего кошелька со включенным доступом к API, который вы желаете использовать в запросах.
  • SecurityToken.Этот параметр состоит из хэшированной строки текста формируемой путем конкатенации пароля доступа к API и UTC даты.

Создание маркера безопасности (токена)

Чтобы создать маркер безопасности, используйте следующую процедуру.

  • Используйте конкатенацию, чтобы получить следующую строку: 
    Пароль Доступа к API:UTC дата в формате YYYYMMDD:UTC время в формате HH
  • Захешируйте строку выше с помощью SHA256. Например, если пароль доступа к API = MyApiPass и UTC дата = 2010-06-01 15:30 (в 24-часовом формате) 
    Объединение параметров: MyApiPass:20100601:15  

    SHA256 хеш предыдущей строки:
    5950F9BDA723ECDF9859AD205412BB44E0DAB16630CA4462490DE64B02305F8D

Безопасность

Сервис OKPAY SOAP API надежно защищен, для того чтобы его могли использовать только авторизованные пользователи OKPAY. Существует три уровня защиты:

  1. Запрашивается имя пользователя API (WalletID) и токен безопасности;
  2. Данные передаются с помощью Secure Sockets Layer (SSL);
  3. Фильтрация доступа к API по IP адресам в настройках кошелька (опционально).

Отсутствие любого из этих уровней влечет отказ в доступе к сервису OKPAY SOAP API.
