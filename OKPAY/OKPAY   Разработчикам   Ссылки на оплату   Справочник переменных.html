Справочник переменных

Используя наше API можно программно создавать кнопки и ссылки, точно такие же как кнопки и ссылки сгенерированные внутри вашего профиля OKPAY.

Сгенерированый код оплаты

Когда вы создаете кнопку, OKPAY генерирует код, который можно вставить в вашу веб-страницу или документ.

Сгенерированный код включает в себя:

  • HTML код для вставки на веб-страницы.
  • URL ссылку для вставки в сообщения электронной почты или документы поддерживающие ссылки.

Определяющие HTML переменные

Когда вы создаете кнопку, вы должны указать те же HTML переменные, которые указываются при генерации кнопок внутри вашего профиля в разделе Кнопки и ссылки оплаты.

В следующей таблице приводится подробная информация об этих переменных:

HTML переменная Обязательное или необязательное поле Описание Максимальная длина

Информация о получателе платежа
ok_receiver Обязательно Один из следующих вариантов: адрес электронной почты, мобильного телефона, или номер кошелька (ID). 127

Информация о платеже
ok_currency Не обязательно Валюта платежа. По умолчанию используется евро. См. раздел Коды валют для списка допустимых значений. 3
ok_invoice Не обязательно Сквозная переменная - номер инвойса или ID транзакции в вашей базе данных, позволяет вам отследить покупку продукта или услуги. Указанное значение будет передано обратно после завершения процесса оплаты (вы сможете его использовать в вашем IPN обработчике или на странице авто возврата). 127
ok_fees Не обязательно Может принимать два значения. “0” – комиссия платится продавцом (по-умолчанию); “1” – комиссия платится покупателем. 1
ok_item_#_name Обязательно Имя этого товара/услуги. 127
ok_item_#_article Не обязательно Сквозная переменная - артикульный номер товара, позволяющая связать покупку с вашей базой данных. Указанное значение будет передано обратно после завершения процесса оплаты (вы сможете его использовать в вашем IPN обработчике или на странице авто возврата). 40
ok_item_#_type Не обязательно Один из типов товара. По-умолчанию digital. 64
ok_item_#_quantity Не обязательно Количество товара, может быть введено клиентом или указано вами. Если больше единицы, значение будет умножать сумму. 8
ok_item_#_price Не обязательно Цена товара. Ограничения: значение должно быть положительным числом. Без символа валюты. Максимум два знака после запятой, разделить целой и дробной части - точка (.) Используйте запятую (,) для дополнительного разделения тысяч.
Общая сумма по каждому элементу будет рассчитываться следующим образом:
ЭлементВсего = ok_item_#_price * ok_item_#_quantity + ok_item_#_tax + ok_item_#_handling + ДоставкаВсего (см. ниже);		 15
ok_item_#_handling Не обязательно Используется только в случае если вы добавляете к цене данного товара стоимость его обработки. 15
ok_item_#_shipping Не обязательно Используется только в случае если вы добавляете к цене данного товара стоимость доставки.
Общая сумма доставки для каждого элемента будет рассчитываться следующим образом:
ДоставкаВсего = ok_item_#_shipping + ok_item_#_shipping2 * (ok_item_#_quantity -1);		 15
ok_item_#_shipping2 Не обязательно Стоимость доставки за каждое дополнительное количество товара. 15
ok_item_#_tax Не обязательно Используется только в случае если вы добавляете к цене данного товара сумму налога. 15
ok_item_#_custom_1_title Не обязательно Указанный вами заголовок настраиваемого поля №1. 127
ok_item_#_custom_1_value Не обязательно Значение настраиваемого поля №1, указанное вами или введенное вашим клиентом. 127
ok_item_#_custom_2_title Не обязательно Указанный вами заголовок настраиваемого поля №2. 127
ok_item_#_custom_2_value Не обязательно Значение настраиваемого поля №2, указанное вами или введенное вашим клиентом. 127

Дополнительные сведения
ok_return_success Не обязательно Адрес, на который браузер плательщика будет перенаправлен после завершения оплаты. Например, URL на вашем сайте который отображает страницу с текстом "Спасибо за ваш платеж". По умолчанию пользователь остается на сайте OKPAY, на странице сообщающей об удачной оплате. 127
ok_return_fail Не обязательно Адрес, на который браузер плательщика будет перенаправлен в случае отмены оплаты. Например, URL на вашем сайте который отображает страницу с текстом "Платеж отменен". По умолчанию пользователь остается на сайте OKPAY, на странице сообщающей об отмене платежа. 127
ok_ipn Не обязательно Переопредение URL адреса (вместо указанного в свойствах кошелька) на который OKPAY передаст информацию об оплате в форме Уведомления об оплате. 127
ok_language Не обязательно Язык страниц логина и регистрации плательщика. Если не указан, язык определяется по кукисам OKPAY в браузере клиента. Если кукисов нет, то язык выбирается автоматически по IP адресу или языку браузера. 2
