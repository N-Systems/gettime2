История IPN операций

История IPN предоставляет дополнительную информацию для устранения возможных проблем связанных с обработкой IPN сообщений. Вы можете использовать историю для определения статуса отправки IPN сообщений, а также для того чтобы в случае необходимости отправить их повторно.

  • Доступ к истории IPN
  • Использование истории IPN
  • Повторная отправка IPN сообщений

Доступ к истории IPN

Войдите в ваш аккаунт на сайте OKPAY, затем выберите Профиль -> Торговля -> Инструменты тестирования -> IPN История.

Использование истории IPN

Используйте историю IPN для просмотра IPN сообщений отправленных OKPAY. Вы можете найти IPN сообщения для конкретного кошелька, диапазона дат, состояния отправки и ID связанной операции.

Результаты поиска содержат следующую информацию:

  • Дата и время когда OKPAY создал это IPN сообщение.
  • Является ли это сообщение IPN оригинальным или повторным, на что указывает звездочка (*) в колонке Дата/Время (создания).
  • ID IPN сообщения присвоенное OKPAY.
  • Текущий статус, может принимать одно из следующих значений:
    • Послано указывает, что OKPAY отправил сообщение на ваш IPN обработчик.
    • Сбой указывает, что OKPAY не получил подтверждения сообщения.
    • В очереди указывает, что OKPAY готов отправить сообщение.
    • Повтор указывает, что сообщение отправляется повторно от 1 до 15 раз и OKPAY продолжает попытки отправить сообщение (каждые 30 минут).
  • Ответ вашего сервера на HTTP POST запрос, который передал IPN сообщение вашему обработчику. Для получения дополнительной информации об этих кодах см. RFC 2616: Hypertext Transfer Protocol -- HTTP/1.1.
  • Если сообщение связано с операцией OKPAY, вы увидите ID этой операции.

Для получения дополнительной информации об IPN сообщении, нажмите на ID сообщения. Появится страница с деталями IPN:

В дополнение к информации отображаемой в списке истории IPN, страница с деталями IPN содержит следующие сведения:

  • Является ли это сообщение IPN оригинальным или повторным (поле Кол-во попыток, у оригинального сообщения здесь 1).
  • Дата/время последней отправки.
  • URL вашего обработчика.
    Примечание: Вы не можете изменить URL, если вы просите OKPAY отправить IPN сообщение повторно, оно отправляется на этот URL.
  • Кол-во попыток, прежде чем сообщение было успешно обработано.
  • Тип IPN сообщения.
  • Все параметры направленные на IPN обработчик.

Повторная отправка IPN сообщений

Вы можете использовать историю IPN, чтобы запросить OKPAY отправить IPN сообщение повторно. С помощью поиска найдите определенные IPN сообщения, а затем нажмите на соответствующую ссылку, чтобы отправить их повторно.

После того как сообщение было отправлено, OKPAY выводит сообщение об этом и обновляет статус.

После отправки сообщения, используется ответ сервера на HTTP POST запрос, чтобы обновить поле "код ответа HTTP". Значение 200 указывает, что сервер успешно принял IPN сообщение. Другие значения обычно указывают на ошибку конфигурации сервера на котором размещен ваш IPN обработчик. Для получения дополнительной информации об этих кодах см. RFC 2616: Hypertext Transfer Protocol -- HTTP/1.1. Если вы не видите код ответа, проверьте, работает ли ваш сервер.
