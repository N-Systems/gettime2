Общие определения API

OKPAY использует следующие принципиальные положения, которые могут повлиять на проектирование ваших бизнес-приложений.

  • Перечисления: Перечисления определяются непосредственно в самом OKPAY API.
  • Поиск и устранение неисправностей: OKPAY API возвращает информацию об элементах, которые вызывают ошибки.
  • Обратная совместимость: OKPAY API сообщает номер версии, чтобы бизнес-приложения имели обратную совместимость если на стороне сервера добавлены новые функции.

Кодировка

OKPAY SOAP API предполагает, что все данные в запросах SOAP в передаются в Unicode, в частности, Unicode (или UCS) Transformation Format, 8-битном виде (UTF-8). В ответах SOAP, сервис всегда возвращает данные в UTF-8.
